Introduction to routers

Routers map URLs to views.

They consist of:

  • A set of URLs to map
  • Methods to trigger when URLs are hit
  • History management
  • Events

Routers handle a users request, based on a url, and trigger certain actions within your application.

Creating a router

var Router = Backbone.Router.extend({

});

Creating an instance:

var router = new Router();

Defining routes

Create a route map

var Router = Backbone.Router.extend({
	routes: {
		"": "index"
	},

	index: function() {
		// do something when index route is triggered
	}
});

Defining routes

The route cascade

var Router = Backbone.Router.extend({
	routes: {
		"about": "about",
		"": "index"
	},

	about: function() {
		// do something when "/about" route is triggered
	},

	index: function() {
		// do something when index route is triggered
	}
});

Defining routes

Urls can be as specific as you need

var Router = Backbone.Router.extend({
	routes: {
		"about/contact": "contact",
		"about": "about",
		"": "index"
	},

	contact: function() {
		// do something when "/about/contact" route is triggered
	},

	about: function() {
		// do something when "/about" route is triggered
	},

	...
	
});

Params

Params pick out individual url segments

var Router = Backbone.Router.extend({
	routes: {
		"customers/:id": "customers",
		"about": "about",
		":page": "page",
		"": "index"
	},

	customers: function(id) {
		// id is "jones" after "/customers/jones" is triggered
	},

	page: function(page) {
		// route to the given top level page
	},

	...

});

Splats

Splats give you the rest of the url to parse manually

var Router = Backbone.Router.extend({
	routes: {
		"search/*query": "search",
		"customers/:id": "customers",
		"about": "about",
		":page": "page",
		"": "index"
	},

	search: function(query) {
		// query is a string containing rest of url
		// e.g. "search/?term=volksvagen&type=hatchback"
		//      query === "?term=volksvagen&type=hatchback"
	},

	...

});

Manually triggering

You can programmatically trigger a route

  • First param defines the new url
  • Second param states whether the route callback should be fired; defaults to false
router.navigate("some/route", true);

Cleaner way of doing window.location.href

History management

To use history management within your app you need to initialise your router

Backbone.history.start();

To initialise using the HTML5 push state api - recommended

Backbone.history.start({ pushState: true });

DOM association

For an existing DOM element:

  • The el attribute defines a selector to match a DOM element
  • this.$el contains a handy jQuery/Zepto reference to it
el: '#listContainer' // e.g. a <ul id="listContainer"x>
render: function() {
	this.$el.html('');
	return this;
}

For new DOM elements:

  • The tagName attribute contains the HTML tag to wrap the view in
  • The className attribute contains class name to add to the element
tagName: 'li',
className: 'listItem'

DOM events

A view can receive DOM events and trigger methods.

events: {
	'click #addButton': 'addSomething',
	'mouseover .menu': 'showMenu'
},
addSomething: function(e) {
	// add something
	// e contains the standard jQuery event object
},
showMenu: function(e) {
	// show the menu
	// e contains the standard jQuery event object
}

Nested views

In some circumstances, you want to nest your views.

A good example is a list of items, nested within a list container.

You can render your items from your main view, as follows: 

// ListItemView render method
render: function() {
	this.$el.html(this.model.get('title'));
	return this;
}

// ListView render method
render: function() {
	this.collection.each(function(item) {
		var item = new ListItemView({ model : item });
		this.$el.append(item.render().el);
	}, this);
	return this;
}

// make new collection
var myList = new List([
	new ListItem({ content : "List item 1"}),
	new ListItem({ content : "List item 2"}),
]);
var myListView = new ListView({ 
	el : '#listContainer',
	collection myList
});
myListView.render();

Auto-rendering

Your views can update automatically when something changes on your model or collection

initialize: function() {
	this.model.on('change', function() {
		this.render();
	}, this);
	return this;
}

Sometimes you only want to update part of your view:

initialize: function() {
	this.model.on('change', function() {
		this.update();
	}, this);
},
update: function() {
	this.$el.find('.name').html(this.model.get('name'));
}

Cleaning up

If you're attaching DOM events (or using any timers) in your view, it's good practise to clean them up when your view is removed, to free up resource.

This is best accomplished by overriding the remove method:

remove: function() {
	// Add cleanup code here 
	// example: unbind events!
	this.model.unbind("change", this.update);

	// always call default remove method in the end.
	return Backbone.View.prototype.remove.apply(this, arguments);
}
